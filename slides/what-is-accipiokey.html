Proposed Solution

What is Accipio Key?

  • Proof of concept
  • Improves user typing experience
  • Draws inspiration from current smartphone technologies
Accipio Key is a proof of concept assistive application which aims to help users type in an easier, faster and less frustrating way by utilizing full-text search technologies to normalize and streamline the typing experience on desktops. This project draws inspiration from current smartphone technologies and it tries to mimic the most useful features as best as possible.

How Does Accipio Key Work?

  1. Runs in the background
  2. Listens for user input
  3. Analyzes input (identifies misspellings, typos, snippets)
  4. Takes appropriate action (correction, completion, generation)
Accipio Key does this by running in the background, listening for user keyboard input and providing text correction, completion, and generation across all applications in a standardized manner. This set of standardized features allow users to quickly and effortlessly navigate between applications and retain the word processing features they are accustomed to use.

System Diagram

Here is a basic system diagram of Accipio Key. When the application starts users can login with a username and password. Having multiple accounts on a computer allows for tailored output for the specific user. Once the application starts several event dispatchers are initialized in order to gather user input, process it and dispatch the events to other components of the application. Some of the core events processed are word events, correction events, completion events and many more. The application listens for these events and distributes them to event handlers which output the event result to the screen. Let's now look at each individual feature individually.

Corrections

  • cansas -> kansas
  • peopl -> people
First let's look at corrections. They are very simple to understand. Accipio Key keeps track of what you have typed and what you are currently typing. When you are done typing a word Accipio Key checks if the word was misspelled. If the word was misspelled it deletes the misspelled word and inputs the correction to the screen.

Completions

  • ca -> (cat, car, castle, camp)
  • ne -> (new, nest)
Completions are slightly different from corrections. Completions process the word which you are currently typing and try to complete the it. This feature requires the press of a shortcut combination to complete the word. Completions will not occur automatically, unless the user specifies.

Generation

  • lol -> laugh out loud
  • brb -> be right back
Generations are similar to completions in that they take the current word you are typing as input. But they differ from completions because there is no guessing on what the user is trying to type. The user defines short form words like 'lol' and 'brb' and assign them text to be generated when they type in the short form word and a keyboard combination.

Underlying Technologies

  • Python 3
  • Kivy (GUI)
  • TextBlob
  • Elasticsearch
  • MongoDB
Before we get to the demo. Let me outline the underlying technologies used in this project. Python 3 was used as the general purpose programming language. Kivy was used for the Graphical user interface. TextBlob and Elasticsearch where used for text processing. Finally MongoDB was used to store user related information.
